O que um entrevistador queria ouvir aqui são termos como **System Call (syscall)**. O programa em *user space* não fala direto com o hardware; ele faz uma chamada de sistema, pedindo ao Kernel (que roda em *kernel space*) para ler o arquivo por ele. O Kernel então usa os drivers para acessar o disco, lê os dados e os entrega de volta para o programa. Esse é o "pulo do gato" que diferencia um conhecimento básico de um conhecimento mais sólido.
